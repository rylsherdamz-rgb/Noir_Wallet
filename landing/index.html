How it works

From tap to settlement, in seconds.

01 · Welcome

Your wallet is your identity.

Link an NFC card, sticker, or wearable — and you're ready to pay. No separate hardware wallet, no seed phrase to memorize on day one.

02 · Dashboard

Every asset, one glance.

PHP, USDC, and XLM balances update live — with per-device wallet limits so you always know what's spendable today.

03 · Link Device

Provision in seconds.

Hold your tag to the phone. You sign once — Soroban's device_registry contract registers it on-chain, permanently.

04 · x402 Agent

Fund it once. It pays for you.

Each device gets its own x402 agent — a dedicated wallet with a balance and a daily limit. It signs every tap, so your main funds stay untouched.

Defense in depth

Simple to use. Robust by design.

Every NFC device links to its own agent wallet with a configurable daily limit. Lose the tag, revoke it instantly — the remaining balance returns automatically to your main wallet.

Hashed Device Identity

Tag UIDs are SHA-256 hashed at the terminal — the raw UID is never stored or transmitted.

Terminal Authentication

Only whitelisted POS terminals can initiate payments, via API keys and signed challenges.

Replay Protection

Every tap carries a signed nonce; 500ms anti-double-tap rate limiting per device.

Blast-Radius Limits

Per-device daily limits and isolated x402 agent wallets — a lost tag never exposes main funds.

Encrypted Key Custody

Fee-channel and agent keys sealed with envelope encryption — a KMS-backed KeyManager.
